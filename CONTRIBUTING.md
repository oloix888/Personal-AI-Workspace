# Contributing

Thank you for helping improve Personal AI Workspace.

## Before opening a pull request

1. Search existing issues and pull requests.
2. For structural changes, open a proposal issue first.
3. Explain the problem, evidence, affected modules, benefits, risks, alternatives and rollback.
4. Preserve original attribution and mark modified files.
5. Do not include personal workspace IDs, private emails, credentials or connector secrets.
6. Run `python scripts/validate_repo.py`.
